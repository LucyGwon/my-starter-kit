# 맛집 큐레이션 리스트 개발 로드맵

`docs/PRD.md`를 기반으로, 구현 우선순위를 **골격 -> 공통 -> 핵심 기능 -> 추가 기능 -> 최적화/배포** 순서로 정리합니다.

## Phase 1. 프로젝트 초기 설정 (골격 구축)

- **예상 소요 시간**: 1~2일
- **할 작업**
  - Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui 초기 세팅
  - 프로젝트 디렉터리 구조 정리(`app`, `components`, `lib`, `types`)
  - 환경변수 구성(`NOTION_API_KEY`, `NOTION_DATABASE_ID`) 및 로딩 검증
  - 기본 레이아웃/글로벌 스타일/에러 페이지 뼈대 구성
  - 라우트 골격 준비(`/`, `/restaurant/[id]`)
- **왜 이 순서인가**
  - 실행 가능한 기본 구조가 먼저 있어야 이후 공통 모듈과 기능을 안정적으로 얹을 수 있습니다.
  - 환경변수/라우팅 골격이 없으면 Notion 연동 및 페이지 개발이 반복적으로 깨질 수 있습니다.
- **완료 기준**
  - 로컬에서 앱이 정상 실행되고 기본 라우트 접근이 가능하다.
  - 환경변수가 주입되어 Notion API 호출 준비가 완료된다.
  - 공통 레이아웃이 전체 페이지에 일관되게 적용된다.

## Phase 2. 공통 모듈/컴포넌트 개발

- **예상 소요 시간**: 2~3일
- **할 작업**
  - Notion API 공통 유틸 구현(클라이언트 초기화, DB 조회, 데이터 변환)
  - 공통 타입 정의(`Restaurant`, `Category`, `RatingFilter`)
  - 공통 UI 컴포넌트 개발(`RestaurantCard`, `FilterBar`, `SectionHeader`, `EmptyState`)
  - 유효성/에러 처리 헬퍼 및 로딩/스켈레톤 컴포넌트 추가
  - 카카오맵 링크 처리 유틸 및 공통 포맷터 작성
- **왜 이 순서인가**
  - 목록/상세/필터 기능이 공통 데이터 모델과 컴포넌트를 공유하므로 먼저 구축해야 중복을 줄일 수 있습니다.
  - 공통 모듈이 선행되면 이후 기능 개발 속도와 품질이 함께 올라갑니다.
- **완료 기준**
  - Notion 응답을 앱 표준 타입으로 안정적으로 변환할 수 있다.
  - 목록/상세 페이지가 공통 컴포넌트 기반으로 조립 가능하다.
  - 공통 에러/로딩 처리 패턴이 확정되어 재사용 가능하다.

## Phase 3. 핵심 기능 개발

- **예상 소요 시간**: 3~4일
- **할 작업**
  - 홈(`/`) 맛집 카드 목록 렌더링 구현
  - 공개 여부(`공개 여부`)가 켜진 데이터만 노출하도록 필터링
  - 카테고리/별점 실시간 필터 기능 구현(MVP 범위)
  - 상세(`/restaurant/[id]`) 페이지 구현(식당 정보, 한줄평, 카카오맵 링크)
  - 빈 상태/잘못된 ID/Notion 응답 실패 예외 처리
- **왜 이 순서인가**
  - PRD의 MVP 핵심 가치(목록 조회 + 상세 확인 + 필터링)를 먼저 완성해야 서비스 본질이 검증됩니다.
  - 핵심 사용자 플로우가 동작한 뒤에야 부가 기능 투자 우선순위를 합리적으로 판단할 수 있습니다.
- **완료 기준**
  - 사용자가 홈에서 필터링 후 상세 페이지까지 이동하는 흐름이 정상 동작한다.
  - 카카오맵 링크가 유효한 경우 정상 이동한다.
  - MVP 범위 기능이 모바일/데스크톱에서 모두 동작한다.

## Phase 4. 추가 기능 개발

- **예상 소요 시간**: 1~2일
- **할 작업**
  - UX 개선(필터 초기화, 상태 유지, 카드 인터랙션 개선)
  - 접근성 보완(키보드 탐색, aria 속성, 색상 대비 점검)
  - PRD 제외 항목의 사전 준비(검색/지도 뷰 확장 고려한 코드 구조 정리)
  - 운영 편의 기능(빈 필드 대체 문구, 링크 안전 처리) 추가
- **왜 이 순서인가**
  - 핵심 기능이 먼저 안정화되어야 UX/확장성 개선이 실제 사용 흐름에 맞춰 적용됩니다.
  - 추가 기능은 핵심 기능을 해치지 않는 범위에서 점진적으로 반영하는 것이 안전합니다.
- **완료 기준**
  - 핵심 흐름을 유지한 상태로 사용성이 눈에 띄게 개선된다.
  - 주요 접근성 체크 항목이 충족된다.
  - 차기 기능(검색/지도) 확장을 위한 구조가 준비된다.

## Phase 5. 최적화 및 배포

- **예상 소요 시간**: 1~2일
- **할 작업**
  - 성능 최적화(이미지, 번들, 데이터 fetch 전략 점검)
  - SEO 기본 설정(title/description/OG, sitemap, robots)
  - 반응형 최종 점검 및 크로스 브라우저 확인
  - Vercel 배포 및 환경변수 설정, 배포 후 스모크 테스트
  - 장애 대응을 위한 최소 로그/모니터링 포인트 정리
- **왜 이 순서인가**
  - 기능 완성 전 최적화/배포를 진행하면 재작업 가능성이 커집니다.
  - 마지막 단계에서 통합 품질을 끌어올리고 운영 안정성을 확보해야 실제 서비스 가치가 완성됩니다.
- **완료 기준**
  - 운영 URL에서 핵심/추가 기능이 모두 정상 동작한다.
  - 성능/SEO 기본 요구사항이 충족된다.
  - 배포 환경변수와 운영 체크리스트가 문서화된다.

## 일정 요약

- **총 예상 기간**: 8~13일
- **진행 원칙**: 각 Phase의 완료 기준 충족 후 다음 단계로 이동
