# PRD: 맛집 큐레이션 리스트

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 맛집 큐레이션 리스트 |
| 목적 | Notion을 CMS로 활용한 개인 맛집 기록 및 공유 웹서비스 |
| CMS 선택 이유 | Notion API를 활용하여 비개발자도 콘텐츠 관리 가능 |

---

## 주요 기능

1. **맛집 목록 조회** — Notion DB에서 맛집 데이터를 불러와 카드형으로 표시
2. **카테고리 / 별점 필터링** — 한식·중식·양식 등 카테고리와 별점으로 실시간 필터
3. **맛집 상세 페이지** — 식당 정보, 한줄평, 카카오맵 링크 제공

---

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| Frontend | Next.js 15, TypeScript |
| CMS | Notion API (`@notionhq/client`) |
| Styling | Tailwind CSS, shadcn/ui |
| Icons | Lucide React |

---

## Notion 데이터베이스 구조

| 필드명 | 타입 | 설명 |
|--------|------|------|
| 식당명 | Title | 식당 이름 (필수) |
| 카테고리 | Select | 한식 / 중식 / 양식 / 일식 / 기타 |
| 별점 | Number | 1 ~ 5점 |
| 위치 | Rich Text | 주소 또는 지역명 |
| 카카오맵 URL | URL | 카카오맵 바로가기 링크 |
| 방문일 | Date | 최근 방문 날짜 |
| 한줄평 | Rich Text | 짧은 리뷰 (100자 이내 권장) |
| 썸네일 | Files & Media | 음식 또는 외관 사진 |
| 공개 여부 | Checkbox | 웹에 노출할지 여부 |

---

## 화면 구성

| 화면 | 경로 | 설명 |
|------|------|------|
| 홈 (목록) | `/` | 카드 그리드 + 카테고리·별점 필터 바 |
| 상세 페이지 | `/restaurant/[id]` | 식당 정보, 한줄평, 카카오맵 링크 버튼 |

---

## MVP 범위

- Notion DB 연동 및 공개 맛집 목록 렌더링
- 카테고리 버튼 필터 (클라이언트 사이드)
- 상세 페이지 (카카오맵 링크 포함)
- 반응형 레이아웃 (모바일 대응)

> **제외 (v2 이후):** 검색, 지도 뷰, 사용자 리뷰 등록, 인증

---

## 구현 단계

1. **Notion 세팅** — DB 생성, Integration 연결, API Key 발급
2. **프로젝트 초기화** — Next.js 15 + TypeScript + Tailwind + shadcn/ui 설치
3. **Notion API 연동** — `@notionhq/client` 설치, DB 쿼리 유틸 작성
4. **목록 페이지 구현** — 카드 컴포넌트, 필터 UI
5. **상세 페이지 구현** — 동적 라우팅, 맛집 정보 렌더링
6. **배포** — Vercel 배포 + 환경변수 설정
